通信プロトコルの概観

ハードウェア

  • WebSocket
    使用場所: シグナリングサーバーとの通信
    特徴:
    • TCP基盤の双方向通信プロトコル
    • 常時接続を維持してメッセージをリアルタイム送信
  • WebRTC (Web Real-Time Communication)
    使用場所: クライアントとホスト間のピアツーピア通信
    特徴:
    • 双方向のP2P通信を可能にするリアルタイムプロトコル
    • STUN/TURN/ICEを用いてNAT越えや接続確立
    • DataChannelを通してデータを低遅延でやり取り

接続と認証

  • STUN（Session Traversal Utilities for NAT）: クライアントのグローバルIPアドレスを取得
  • TURN（Traversal Using Relays around NAT）: P2P接続が困難な場合に中継サーバーを使用
  • ICE（Interactive Connectivity Establishment）: 複数の候補接続を評価し、最適な通信経路を確立
  • 認証: ed25519 + base64署名
    • クライアントは秘密鍵で認証コードを署名し送信
    • サーバーは公開鍵で検証

データ伝送

DataChannel (WebRTC): クライアントとホストの操作通信に使用。低遅延・高効率な通信手段。

通信プロトコル形式（データのヘッダー）

ヘッダー内容形式例
mmマウス移動mm{x},{y}
mpマウスポインターの位置設定mp{x},{y}
mdマウスドラッグmd{x},{y}
msスクロールms{x},{y}
muマウスボタンUPmu
mbマウスボタン押下mb{type}
kpキー入力（押下）kp{key}
kdキー押し続けkd{key}
kuキー離すku{key}
ca認証コード要求ca{code}
cb認証コード応答（署名）cb{signature}

WebSocketメッセージ構造（JSON）

{
  "mtype": "sdp", 
  "tohost": "abc123",
  "body": { ... },
  "fromhost": "xyz789"
}
  • mtype: メッセージの種類（例: sdp, ice, hosts, sdpoffer）
  • tohost/fromhost: 接続先/送信元識別
  • body: 実際の内容（SDP, ICE候補, hostリストなど）

セキュリティ

ed25519 + base64による認証: クライアントが送信するca要求に対して、秘密鍵を使った署名でcbを返答

フォールバック

予備シグナリングサーバー: sec_sigserver として接続不能時の代替を用意

このシステムは、マウス・キーボードなどのリモート操作をWebRTC経由でリアルタイムに行うための通信プロトコルを実装しています。データはDataChannelで送信され、セキュアな認証方式により改竄やなりすましを防止します。
